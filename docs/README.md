# Docs Folder

This folder contains all reports, profiles, planning documents, and weekly logbooks for the SmartTimetable project, organized roughly by the lab in which each was produced.

## Files

| File | Lab | Description |
|---|---|---|
| `team-profile.md` | Lab 1 | Team name, member names, roles, and responsibilities |
| `idea-log.md` | Lab 1 | Initial log of candidate IT venture ideas considered before opportunity scanning |
| `opportunity-scan.md` | Lab 2 | Scan and scoring of possible IT opportunities using the NUF (New, Useful, Feasible) framework |
| `selected-opportunity.md` | Lab 2 | Writeup of the selected opportunity — SmartTimetable — and why it was chosen over the alternatives |
| `customer-questions.md` | Lab 3 | Survey/interview questions used for customer discovery |
| `problem-notes.md` | Lab 3 | Detailed breakdown of target respondents, pain points, workarounds, and consequences from customer discovery |
| `assumption-evidence-table.md` | Lab 3 | Compares the team's original assumptions about the timetable problem against actual survey evidence |
| `customer-discovery-summary.md` | Lab 3 | High-level summary of survey findings, strongest/weakest evidence, and the decision to continue with SmartTimetable |
| `user-persona.md` | Lab 4 | Primary user persona (Yu), including background, goals, pain points, and design implications |
| `user-stories.md` | Lab 4 | User stories for Student, Lecturer, and Admin actors, mapped to functional requirements |
| `mvp-feature-list.md` | Lab 4 | MVP feature list prioritized using MoSCoW |
| `system-requirements.md` | Lab 4 | Full functional requirements list (FR-01 through FR-16) |
| `product-concept.md` | Lab 5 | Short product concept statement: what SmartTimetable is, who it's for, and what problem it solves |
| `feature-requirement-mapping.md` | Lab 5 | Maps each required screen to its functional requirement IDs |
| `wireframe-specification.md` | Lab 5 | Screen-by-screen specification for every required and supporting wireframe/mockup |
| `wireframe-usability-review-checklist.md` | Lab 5 | Usability review checklist applied to the wireframes before moving to the prototype |
| `screen-inventory-checklist.md` | Lab 5 | Tracks which required screens exist, whether they have a wireframe/screenshot, and any notes |
| `business-model-canva.md` | Lab 6 | Full business model canvas (customer segments, value proposition, channels, revenue/sustainability logic, key resources, activities, partners, and costs) |
| `feature-value-mapping.md` | Lab 6 | Maps each MVP feature to its requirement ID, user story, user value, business value, and priority |
| `technical-architecture.md` | Lab 6 | Selected prototype platform, main components, what will be fully implemented vs. simulated, and the biggest technical risk |
| `data-structure.md` | Lab 6 | Data entities, field definitions, status values, and data privacy notes |
| `weekly-logbook.md` | All labs | Running log of what was completed, decisions made, problems found, and next steps for each lab session |

## Notes
- This README should be updated whenever a doc is added, removed, or renamed
- `technical-architecture.md` and `data-structure.md` are listed here for completeness but may not be pushed to the repo yet — check the file list above against what's actually committed
