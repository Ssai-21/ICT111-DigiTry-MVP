# Product Concept

## 1. Project Title
SmartTimetable – Smart Campus Timetable System

## 2. Target User
Primary users: students and lecturers who need to check their class schedules.
Secondary user: academic staff/admin who maintain and update timetable data.

## 3. Problem Statement
Students and lecturers currently rely on a static PDF timetable that does not update when a class is moved, cancelled, or rescheduled. The PDF is difficult to search on a phone, gives no personalized daily view, and last-minute changes are only communicated informally through group chats or verbal announcements in class.

## 4. Proposed IT Product
A web-based timetable system connected with LINE Notify that helps users view personalized daily/weekly schedules, search by student, lecturer, or room, view class details, and allow an admin to update schedule entries, with automatic LINE notifications sent when changes occur.

## 5. Core Value Proposition
The system reduces confusion and makes timetable information personalized, searchable, and instantly updatable, replacing outdated PDFs and scattered informal announcements.

## 6. MVP Scope
The MVP will focus on timetable data digitization, personalized schedule view, search/filter, item detail view, admin schedule update, LINE Notify alerts, and dashboard summary.

## 7. Out of Scope for Current Semester
- AI-based automatic conflict detection
- Native mobile app deployment
- Real-time push notifications outside LINE
- Integration with official university login/authentication systems
- Advanced role-based access control
